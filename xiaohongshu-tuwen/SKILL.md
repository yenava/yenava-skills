# Xiaohongshu Tuwen

你的目标不是“写几段文案”，而是把内容变成一套可直接发布的小红书图文图片。默认先完成导演稿，再把每一页生成为 HTML/CSS，并渲染成图片。

## 核心判断

这类图文的本质不是海报，而是“媒体化信息卡片”：

- 每一页只表达一个核心判断
- 用图表、数据卡、对比卡替代长段落
- 用统一视觉系统把多页内容串成一个系列
- 读者 3 秒内先看到结论，10 秒内看懂证据

先读取 [references/reverse-engineering.md](references/reverse-engineering.md) 掌握这种风格的视觉和内容机制。  
需要决定整套图文的视觉主题时，读取 [references/theme-templates.md](references/theme-templates.md)。  
需要设计具体页面时，再读取 [references/page-recipes.md](references/page-recipes.md)。  
输出最终交付物时，遵循 [references/output-contract.md](references/output-contract.md)。  
需要实际出图时，遵循 [references/html-rendering.md](references/html-rendering.md)。  
所有页面在截图前，还要遵循 [references/layout-qa.md](references/layout-qa.md) 做布局体检。

## Workflow

按下面顺序执行。

### Step 1: 锁定创作 Brief

先判断以下 4 项是否明确：

1. 主题是什么
2. 想让读者记住什么核心观点
3. 证据类型是什么
4. 信息量大概有多少层

如果用户没说完整，不要卡住。优先做合理假设，并在开头写明：

- 默认平台：小红书竖版轮播
- 默认页数：根据内容量自动决定
- 默认赛道：科技 / 商业 / 职场 / 趋势解读
- 默认目标：让读者快速看懂一个变化或一个判断

### Step 2: 先定“信息骨架”，再写文案

先把内容归入下面 4 类之一：

- 观点判断型：某件事意味着什么，适合封面 + 证据 + 启示
- 数据趋势型：某个趋势怎么变了，适合折线图 / KPI 卡 / 结论页
- 对比拆解型：两个对象有什么差异，适合对比表 / 双列卡片
- 盘点总结型：某个领域发生了什么，适合列表 + 时间线 + 归纳页

然后先统计“信息块”数量，再决定页数。信息块通常包括：

- 一个封面问题或主判断
- 一个核心数据总览
- 一个对比关系
- 一个趋势变化
- 一个原因拆解
- 一个影响或启示
- 一个结论或行动建议

页数决定规则：

- 2 到 3 个信息块：做 3 到 4 页
- 4 到 5 个信息块：做 5 到 6 页
- 6 到 7 个信息块：做 7 到 8 页
- 超过 7 个信息块：合并弱信息，通常不要超过 9 页

然后决定页面结构。默认优先从下面模块里选，不要求全部使用：

1. 封面页
2. 核心数据页
3. 对比或分解页
4. 趋势 / 原因页
5. 启示页
6. 结论或行动页

不要先假设固定 6 页。先出页纲，再定最终页数。

### Step 3: 先选主题模板，再建立统一视觉系统

不要一上来就默认同一套暖白橙红风格。先根据内容自动选主题模板，再做该模板下的视觉统一。

先判断以下 5 个信号：

1. 主要任务是解释、对比、预警、盘点还是给方法
2. 证据以数据、案例、步骤还是观点为主
3. 语气更像新闻编辑部、研究简报、行动手册还是高端品牌观察
4. 读者更想快速看结论，还是愿意顺着卡片一步步学
5. 这个主题需要“稳重可信”还是“强提醒感”

然后读取 [references/theme-templates.md](references/theme-templates.md)，从中选 1 套主模板。

选择原则：

- 数据、趋势、行业解读优先选 `data-newsroom`
- 排名、优劣、横向比较优先选 `benchmark-board`
- 教程、方法论、步骤拆解优先选 `playbook-cards`
- 概念解释、模型拆解、知识科普优先选 `explainer-notebook`
- 强变化、反常识、风险提醒优先选 `signal-alert`
- 品牌案例、公司气质、审美型商业观察优先选 `premium-brief`

如果内容同时跨两类，只允许：

- 选 1 套主模板
- 借用 1 种辅助模块语言

不要把 3 套以上风格混在同一轮播里。

沿用参考图的“轻杂志感”视觉，不要做成营销海报。模板可以变，但系列感不能散。

强制执行：

- 画布优先 3:4 竖版，推荐 1080x1440
- 背景用暖白或纸感浅米色
- 主文字深黑或深灰，强调色只保留 1 个主色
- 模块以圆角卡片、细描边、轻阴影来组织
- 标题区必须有清晰层级，关键词可单独高亮
- 每页底部保留数据来源或注释区

如果用户没有指定品牌视觉，默认用主模板对应的色板和组件语言。只有在主模板没有明显约束时，再回落到默认方案：

- 背景：暖白 / 米白
- 主色：橙红
- 辅色：深蓝灰
- 点缀：浅金 / 浅灰

### Step 4: 逐页生成“导演稿”

每页必须包含：

- 主题模板：这一页沿用哪套主模板
- 页面角色：这一页负责讲什么
- 页面标题：适合直接放到画面上的标题
- 核心信息：1 到 3 个要点
- 图形建议：该页适合什么图表或卡片
- 版式指令：模块怎么摆，重点落在哪里
- 文案限制：避免超过画面承载量

优先让数字、标签、对比关系上画面，不要把解释都塞成段落。

### Step 5: 输出时要“可执行”

最终交付分两层。

第一层是内容和排版规划，默认包含：

1. 选题角度
2. 已选主题模板与选择理由
3. 整体视觉设定
4. 页面结构总览
5. 逐页导演稿
6. 封面标题备选
7. 可选的发布文案

第二层是实际图片生成。只要用户要最终图片，就继续执行：

1. 为每一页生成结构化 HTML
2. 为整套轮播生成共享 CSS，而不是每页各写一套内联样式
3. 统一所有页面的背景、色板、字重、圆角、边距和组件语言，并落实到所选主模板
4. 用浏览器渲染 HTML 页面，再截图导出 PNG
5. 对每一页检查标题可读性、数字清晰度和系列一致性
6. 将最终 HTML、CSS 和 PNG 一起保存到工作区，默认放在 `output/xiaohongshu-tuwen/<topic-slug>/`

如果用户明确说要“给设计师交付”或“只要静态页面”，再追加对应版本：

- Figma / Canva 排版说明版
- HTML 组件说明版
- 口播 / 配文联动版

### Step 6: 逐页出图

生成完导演稿后，不要停。继续为每页执行 HTML/CSS 出图。

出图原则：

- 一页一个 HTML 文件，保证版式和文字可控
- 整套至少共享一个 `styles.css`
- 所有页使用同一套主色、边距、卡片圆角和标题层级
- 图文是代码排版结果，不是模型脑补海报
- 所有页面都要保留编辑感、数据感、商业杂志感
- 页面里的标题、数字、短标签必须是可编辑文本，不要烘焙成图片背景
- 每页可以换版式模块，但不能脱离主模板的视觉母体

每页 HTML 至少包含：

- 页码与页面角色
- 根容器：1080x1440 竖版
- 顶部信息条：左侧为栏目或页码，右侧为可选作者名
- 页面标题和副标题
- 主要图表类型对应的模块结构
- 必须出现的数字和短标签
- 页底来源或注释区
- 对应的语义类名，方便 CSS 统一控制

推荐额外标注：

- 主要内容模块加 `data-block`
- 装饰元素加 `data-decorative="true"`

生成后检查：

- 标题是否能一眼看清
- 数字是否比说明文字更突出
- 同一套轮播的视觉是否一致
- 页底是否预留来源或注释位置
- 是否存在明显的底部空洞
- 是否存在内容块重叠
- 是否把正文内容错误地做成绝对定位

如果某一页排版明显失衡，优先重写该页 HTML 或局部 CSS，不要整套推倒重来。

### Step 7: 布局 QA 后再截图

在最终截图前，必须先跑布局检查。

默认执行：

```bash
node scripts/check-layout.mjs output/xiaohongshu-tuwen/<topic-slug>
node scripts/render-pages.mjs output/xiaohongshu-tuwen/<topic-slug>
```

如果布局检查报错，不要直接交付 PNG。先修 HTML/CSS，再重新检查。

## 写作要求

- 标题短、结论强，少用空泛套话
- 每页只放一个主判断，不要一页塞五层逻辑
- 数字优先大字号展示，说明文字退到辅助层
- 强调“变化”“差异”“代价”“结论”，而不是泛泛介绍
- 如果数据未经验证，要明确标注“预测”“估算”或“公开口径”
- 如果用户给的是新闻事件，不要写成新闻摘要，要写成“观点化信息图文”
- 每次都明确写出本次选中的主题模板和选择理由，避免风格看起来像随机抽卡

## 禁止事项

- 不要输出只有文案、没有页面结构的结果
- 不要默认使用夸张网感标题党
- 不要在右上角默认写死媒体名或品牌名；该位置是作者名，用户没提供就留空
- 不要满页渐变、满页装饰、满页图标
- 不要把所有页面做成同一种模板
- 不要把图表当装饰，图表必须服务于结论
- 不要把正文卡片大规模用 `position: absolute` 硬摆出来
- 不要在封面页或总结页留下明显失控的大块空白
- 不要在未通过布局检查前直接截图交付

## 默认响应姿势

当用户只说“帮我做一套这种风格的小红书图文”时，默认按以下顺序回复：

1. 用一句话复述主题和主判断
2. 判断内容类型并自动选 1 套主题模板
3. 根据内容量自动决定页数，并给出整套图文的页面骨架
4. 给出统一视觉方向
5. 逐页输出导演稿
6. 逐页生成 HTML/CSS 并渲染最终图片
7. 最后补 3 个封面标题备选
