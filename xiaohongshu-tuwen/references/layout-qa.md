# Layout QA

## 目录

- 默认目标
- 内容排版原则
- 页面结构风险
- 检查方式
- 常见修复动作
- 交付门槛

这套 skill 不只负责“生成页面”，还要负责“把页面排对”。最终截图前，必须做一次布局体检。

## 一、默认目标

每一页都至少满足这 4 条：

- 没有内容块互相重叠
- 没有明显超出画布的内容
- 没有大块非刻意留白
- 装饰元素不会压住正文信息

如果一页不能满足这 4 条，就不算完成。

## 二、内容排版原则

### 1. 正文内容优先流式布局

优先使用：

- Grid
- Flex
- 正常文档流中的 section / article / card

避免把正文内容做成大量绝对定位。  
`position: absolute` 默认只允许用于：

- 背景字母
- 装饰线条
- 弱化图形底纹
- 极少量经过验证的标号徽章

不要用绝对定位去摆正文卡片、标题、说明段落。

### 2. 页面要“吃满”画面，但不能挤爆

经验规则：

- 正文最底部与页脚之间不应出现过大的空洞
- 封面页如果下半区明显空，应补一个结论区、趋势区或摘要区
- 如果内容过满，不要硬塞，优先拆页

默认把“异常大留白”理解为设计问题，而不是留白美学。

### 3. 复杂机制页优先矩阵，不优先漂浮结构

当一页需要放 4 个以上步骤、模块或节点时：

- 优先用 2 列或 3 列矩阵
- 优先用时间线 / 步骤卡 / 表格
- 不要默认做漂浮环形、漂浮气泡、自由散点卡片

只有在验证过不重叠、不拥挤时，才允许更激进的布局。

## 三、标注规范

为了让检查脚本更稳，建议：

- 主要内容模块加 `data-block`
- 装饰元素加 `data-decorative="true"`

示例：

```html
<section class="cover-bottom" data-block>
  ...
</section>

<div class="hero-letter" data-decorative="true">H</div>
```

## 四、检查流程

在截图前，先运行启发式布局检查：

```bash
node scripts/check-layout.mjs output/xiaohongshu-tuwen/<topic-slug>
```

默认会检查高风险问题：

- 正文绝对定位
- 漂浮卡片类高风险结构
- 封面空洞风险
- 内容块数量过少导致的空页风险

如果你用 `node scripts/render-pages.mjs ...`，它会先自动跑这一步，再开始截图。

## 五、发现问题时怎么修

### 情况 A：底部留白太大

- 增加一个信息承接模块
- 改成上下两段结构
- 如果这页信息实在太少，考虑与前后页合并

### 情况 B：元素重叠

- 回到流式布局
- 改成 Grid / Flex
- 减少同页信息块数量
- 放弃漂浮式卡片

### 情况 C：标题太挤

- 缩短标题
- 手动断行
- 增加标题区高度

### 情况 D：画面太满

- 删掉辅助信息
- 把长句改成短标签
- 拆到下一页
