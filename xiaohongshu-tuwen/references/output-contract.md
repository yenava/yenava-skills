# Output Contract

## 目录

- 标准输出格式
- 输出细则
- 当用户要“更像设计稿”时
- 当用户要“更像代码稿”时
- 当用户要“直接出图”时

默认把结果写成“可执行导演稿 + HTML/CSS 交付说明 + 最终图片交付说明”，不要只给散文式建议。

## 输出细则

- 页面标题优先短句，尽量控制在 8 到 18 个字
- 每页核心信息控制在 1 到 3 个点
- 页数必须根据内容量决定，不能机械固定成 6 页
- 先声明本次选中的主题模板，再展开视觉设定
- 如果是数据页，要明确数据口径或来源位置
- 如果是对比页，要明确“胜负手”维度，不要平均铺陈
- 如果是趋势页，要点名最关键的转折点
- 输出最终图片时，必须同时交付 HTML 和 CSS

## 当用户要“更像设计稿”时

把每页补成更细的排版说明：

- 顶部区域放什么
- 中部主视觉是什么
- 左右栏分别承载什么
- 哪个数字最大
- 哪个元素需要用强调色

## 当用户要“更像代码稿”时

把每页补成结构化页面说明，至少包含：

- 页面类型
- 根容器 class
- 模块树
- 共享组件 class
- CSS 变量
- 主要图表实现方式

## 当用户要“直接出图”时

在导演稿之后继续执行 HTML/CSS 生成和渲染，并补充：

- 每页对应的 HTML 文件名
- 共享 CSS 文件名
- 每页目标文件名
- 统一的系列视觉变量
- 生成后需要检查的版面清单
